# Engineering Pilot Playbook

## Recommended starting workflow

Pull request drafting and review acceleration for low- to medium-complexity changes.

## Why this is a good pilot

- high frequency
- measurable cycle times
- existing systems usually contain the needed data
- strong opportunity to pair speed with quality safeguards

## Suggested metrics

- weekly active assisted engineers
- median PR cycle time
- review turnaround time
- escaped defect rate
- override or discard rate on suggestions
- percent of AI-assisted changes reviewed before merge

## Recommended systems

- GitHub
- Jira or Linear
- CI status checks

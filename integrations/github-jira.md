# GitHub + Jira Integration

## Best for

Engineering workflows such as coding assistance, pull request creation, code review, and issue-to-release flow.

## Source-of-truth mapping

- workflow intake: Jira issue creation and transitions
- work execution: GitHub commits, pull requests, reviews
- quality: CI results, incident links, defect records
- governance: review coverage, policy checks, security scans

## Useful events

- issue moved to in progress
- pull request opened
- AI suggestion accepted or discarded
- review requested
- review completed
- pull request merged
- defect linked after release

## Minimum viable scorecard

- adoption: percent of eligible engineers using AI weekly
- speed: PR cycle time
- quality: escaped defect rate
- throughput: merged PRs per engineer
- leverage: low-complexity tasks completed without escalation
- risk: percent reviewed and policy-compliant before merge

# Metric Review Policy

Metrics should be reviewed as operating tools, not permanent truths.

## Review cadence

- weekly for active pilots
- monthly for scaled workflows
- quarterly for portfolio-level metric retirement or redesign

## Replace a metric when

- it no longer changes decisions
- it is being gamed
- it creates confusion without insight
- the workflow has evolved and the metric no longer fits

## Never allow

- a standalone speed metric without quality context
- a standalone adoption metric presented as business value
- executive reporting built on self-reported savings alone
